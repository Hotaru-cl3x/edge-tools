実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 固定テーブル実装（31system-tool.js）
var TABLE = [
    {stage:'A',bet:1},{stage:'A',bet:1},{stage:'A',bet:1},
    {stage:'B',bet:2},{stage:'B',bet:2},
    {stage:'C',bet:4},{stage:'C',bet:4},
    {stage:'D',bet:8},{stage:'D',bet:8}
];
// 合計: 3×1 + 2×2 + 2×4 + 2×8 = 31単位

S = { pos:0, lastWin:false, ... }

if (r === 'win') {
    S.pnl += ba;
    if (S.lastWin) {
        S.done = true;  // 2連勝 → システム終了
        return;
    } else {
        S.lastWin = true;
        // 次ステージの先頭にジャンプ
        var nextPos = S.pos + 1;
        while (nextPos < TABLE.length &&
               TABLE[nextPos].stage === curStage) {
            nextPos++;  // 同ステージのスロットをスキップ
        }
        S.pos = nextPos;
    }
} else {
    S.pnl -= ba;
    S.lastWin = false;
    S.pos++;  // 次のスロットへ
    if (S.pos >= TABLE.length) { S.done = true; }  // 全消化
}

数式・計算仕様

テーブル総和3×1 + 2×2 + 2×4 + 2×8 = 31単位
最大損失（全消化）31単位（固定・上限保証）
2連勝の判定lastWin フラグで直前の結果を追跡
ステージジャンプ同ステージの残スロットをスキップして次ステージ先頭へ
連勝通知lastWin === true の時「1勝中（次で2連勝！）」表示

メリット・デメリット

✓ メリット

  • 最大損失が31単位に固定されリスクが明確
  • テーブルを配列で実装しスロット進行を直感的に管理
  • 「1勝中」表示で次の2連勝チャンスを把握できる

✗ デメリット・注意点

  • ルールが複雑で手動計算は困難
  • 2連勝が達成できない場合、最大31単位を失う
