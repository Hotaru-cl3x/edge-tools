Oscar's Grind Edge

オスカーズグラインド法をリアルタイム自動計算するツール。1セッション+1単位の利益を目標とし、勝ちでベット増・負けでベット据え置きという保守的なルール。収支±0回復でもシステム終了。

基本仕様

システム種別Positive Progression（保守型）
対象ゲームバカラ・ルーレット（2倍配当）
目標利益+1単位 / セッション
リスクLow
複雑度Medium

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 状態オブジェクト（oscarsgrind-tool.js）
S = { u, m, c, bet:1, rd, pnl, sessionPnl:0, hist, done }

// 勝敗処理（2つの終了条件）
if (r === 'win') {
    S.pnl += ba; S.sessionPnl += ba;

    if (S.sessionPnl >= S.u) {
        S.done = true;  // 終了条件A: +1単位以上の利益
        return;
    } else if (S.sessionPnl >= 0) {
        S.done = true;  // 終了条件B: 収支±0回復
        return;
    } else {
        S.bet = S.bet + 1;  // 通常: ベット+1単位
    }
} else {
    S.pnl -= ba; S.sessionPnl -= ba;
    // 負けはベット据え置き（変更なし）
}

数式・計算仕様

目標利益+1単位 / セッション
終了条件AsessionPnl >= 1単位
終了条件BsessionPnl >= 0（損失回収）
負け時のベット変化0（据え置き）
勝ち時のベット変化+1単位

メリット・デメリット

✓ メリット

  • 負け時にベットが上がらない保守的設計
  • sessionPnlとpnlの分離管理で正確な判定
  • 2種類の終了条件で柔軟な出口戦略

✗ デメリット・注意点

  • 1セッションの利益目標が小さい（+1単位）
  • 長期連敗ではsessionPnlが大きく負に傾く
