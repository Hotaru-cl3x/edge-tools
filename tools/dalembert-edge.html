Negative Progression

D'Alembert Edge

ダランベール法をリアルタイム自動計算するツール。負けで+1単位・勝ちで-1単位という緩やかな変動が特徴。開始単位数を1〜8で設定でき、ベット単位が0になった時点でシステム完了。

ツールを無料で使う →

基本仕様

システム種別Negative Progression（逆張り型）
対象ゲームバカラ・ルーレット（2倍配当）
初期ベット設定可能（1〜8単位、デフォルト5単位）
リスクLow〜Medium
複雑度Low

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 状態オブジェクト（dalembert-tool.js）
S = { u, m, c, bet:start, start:start, rd, pnl, hist, done, cleared:false }

// 勝敗処理
if (r === 'win') {
    S.pnl += ba;
    if (S.bet <= 1) {
        S.bet = 0; S.cleared = true;  // システム完了
    } else {
        S.bet = S.bet - 1;  // -1単位
    }
} else {
    S.pnl -= ba;
    S.bet = S.bet + 1;  // +1単位
}

// UIカラー: 開始単位より上→赤、下→緑
betEl.className = 'sval ' + (S.bet > S.start ? 'n' : S.bet < S.start ? 'p' : '');

数式・計算仕様

N連敗後のベット単位start + N 単位
システム完了条件bet が0になる（勝ち越しが必要）
理論的回収条件勝ち数 = 負け数 + start 回
マーチンゲールとの比較増加ペースが線形（vs 指数関数）

メリット・デメリット

✓ メリット

  • ベット増加が線形でマーチンゲールより安全
  • ベット単位の色による直感的なUI
  • bet=0という明確なシステム完了条件

✗ デメリット・注意点

  • 損失回収には勝ち数が負け数＋start回を超える必要がある
  • 長期連敗では資金が線形に減少
