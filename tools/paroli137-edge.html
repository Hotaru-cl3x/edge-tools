Paroli137 Edge

パロリーシステム137法をリアルタイム自動計算するツール（paroli137-tool.js）。1→3→7単位という3ステップのラダーをSTEPS=[1,3,7]配列で実装。3連勝でサイクル完了・自動リセット。10連敗後でも1サイクル（3連勝）で利益転換できる高い防御力が特徴。

ツールを無料で使う →

基本仕様

システム種別Positive Progression / Fixed 3-step
対象ゲームバカラ・ルーレット（2倍配当）
ステップ配列STEPS = [1, 3, 7]
サイクル3連勝で自動リセット
リスクLow〜Medium

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 3ステップサイクル（paroli137-tool.js）
var STEPS = [1, 3, 7];
S = { si:0, ... }

if (r === 'win') {
    S.pnl += ba;
    S.si++;
    if (S.si >= STEPS.length) {
        // 3連勝完了 → 自動リセット
        S.si = 0;
        S.hist.unshift({ ..., complete: true });
        draw(); return;  // サイクル完了を履歴に記録
    }
} else {
    S.pnl -= ba;
    S.si = 0;  // 1単位にリセット
}

var ba = STEPS[S.si] * S.u;
// si=0: 1単位, si=1: 3単位, si=2: 7単位（ラスト）

数式・計算仕様

ステップ配列STEPS = [1, 3, 7]
1サイクル総投資1 + 3 + 7 = 11単位
1サイクル総獲得（3連勝）1 + 3 + 7 = 11単位（収支±0）
待機中コスト1単位 / 回（最小限）
損益分岐の例10連敗(-10u) + 1サイクル完了(+11u) = +1u

メリット・デメリット

✓ メリット

  • 3ステップのシンプルな実装（配列1行）
  • サイクル完了で自動リセットする明確な終了条件
  • 負け中は常に1単位のため長期連敗でも損失が線形

✗ デメリット・注意点

  • 3連勝の確率は(0.5)^3 ≈ 12.5%
  • 待機ベット（1単位）の連敗が続くと損失が累積
