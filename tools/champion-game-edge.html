Sequence-based / Multi-set

Champion Game Edge

チャンピオンゲーム法をリアルタイム自動計算するツール（champion-tool.js）。予選→決勝の2ステージ×2セット構成。1セット完走で+5単位、2セット完走で+10単位の利益を目指す独自マルチセット設計。

基本仕様

システム種別Sequence-based / Multi-set
対象ゲームバカラ・ルーレット（2倍配当）
構成Set 1（予選+決勝）→ Set 2（予選+決勝）
各セット目標Set1: +5単位 / 累計: +10単位
複雑度Medium

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// マルチセット管理（champion-tool.js）
S = { set:1, phase:'qualify', seq:[1,1], ... }

// 数列クリア時のフェーズ遷移
if (S.seq.length === 0) {
    if (S.phase === 'qualify') {
        S.phase = 'final';
        S.seq = [1, 1, 1];  // 決勝数列（3要素）
    } else if (S.phase === 'final') {
        if (S.set === 1) {
            S.phase = 'set_clear';  // Set1完了 → +5単位
        } else {
            S.phase = 'all_clear';  // 全完了 → +10単位
            S.done = true;
        }
    }
}

// Set2への移行
function nextSet() {
    S.set = 2; S.phase = 'qualify'; S.seq = [1, 1];
}

// ベット計算（モンテカルロと同じ両端の和）
function bet() { return S.seq[0] + S.seq[S.seq.length-1]; }

数式・計算仕様

予選初期数列[1, 1]（ベット: 1+1 = 2単位）
決勝初期数列[1, 1, 1]（ベット: 1+1 = 2単位）
Set1完了時の利益+5単位（理論値）
Set2完了時の累計+10単位（理論値）
進捗計算gained / (u × 10) × 100%

メリット・デメリット

✓ メリット

  • +5/+10単位という明確な段階的目標
  • set/phaseの2軸でゲーム状態を明確に管理
  • マルチセット構造で自然な休憩ポイントを設定

✗ デメリット・注意点

  • セッションが長くなる可能性がある
  • 連敗時に数列が長くなるリスクはモンテカルロと同様
