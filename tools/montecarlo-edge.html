Montecarlo Edge

モンテカルロ法（キャンセレーション法）をリアルタイム自動計算するツール。2倍・3倍配当に対応した配当タイプ切替を搭載。勝ちで消去する両端の組数が配当タイプによって異なる独自実装。

基本仕様

システム種別Sequence-based / Negative Progression
対象ゲームバカラ・ルーレット（2倍・3倍配当、切替可能）
初期数列[1, 2, 3]（固定）
リスクMedium
複雑度Medium

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 状態オブジェクト（montecarlo-tool.js）
S = { u, m, c, p:2or3, seq:[1,2,3], rd, pnl, hist, done }

// ベット計算
function bet() {
    var q = S.seq;
    return q.length === 0 ? 0 :
           q.length === 1 ? q[0] :
           q[0] + q[q.length - 1];  // 左端 + 右端
}

// 勝敗処理（配当タイプで消去数が異なる）
if (r === 'win') {
    S.pnl += S.p === 2 ? ba : ba * 2;  // 3倍は利益2倍
    var n = S.p === 2 ? 1 : 2;         // 消去組数
    for (var i = 0; i < n; i++) {
        if (S.seq.length > 1) { S.seq.shift(); S.seq.pop(); }
        else if (S.seq.length === 1) { S.seq = []; break; }
    }
} else {
    S.pnl -= ba;
    S.seq.push(b);  // 負けたベット単位数を末尾に追加
}

数式・計算仕様

ベット額seq[0] + seq[末尾] 単位
勝ち時の消去数（2倍）両端1組（2要素）を削除
勝ち時の消去数（3倍）両端2組（4要素）を削除
負け時の追加今回のベット単位数をpush
システム完了seq.length === 0

メリット・デメリット

✓ メリット

  • 2倍・3倍配当に対応した汎用実装
  • 数列の縮小でプログレスが把握しやすい
  • 損切りライン自動停止

✗ デメリット・注意点

  • 連敗で数列が長くなりベット額が増大
  • 初期数列[1,2,3]は固定（カスタム不可）
