2in1 Edge

独自開発の2フェーズ制ベッティングシステム（2in1-tool.js）。待機フェーズで2連敗を検出後、初期数列[1,1]でキャンセレーション法を稼働。数列クリアで2回分の損失を一括回収する設計。

基本仕様

システム種別Original / Hybrid Sequence-based
対象ゲームバカラ・ルーレット（2倍配当）
フェーズ待機（1単位固定）→ 稼働（数列法）
初期数列[1, 1]（2連敗時に自動設定）

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 2フェーズ制ロジック（2in1-tool.js）
S = { phase:'wait', consecutive:0, seq:[], ... }

// ベット計算（両端の和）
function bet() {
    if (S.phase === 'wait') return 1;
    var q = S.seq;
    if (q.length === 0) return 0;
    if (q.length === 1) return q[0];
    return q[0] + q[q.length - 1];
}

// 待機フェーズ
if (S.phase === 'wait') {
    if (r === 'lose') {
        S.pnl -= ba; S.consecutive++;
        if (S.consecutive >= 2) {
            S.seq = [1, 1];    // 初期数列
            S.phase = 'active';
            S.consecutive = 0;
        }
    } else { S.pnl += ba; S.consecutive = 0; }
}

// 稼働フェーズ（キャンセレーション）
if (r === 'win') {
    S.pnl += ba;
    if (S.seq.length > 1) { S.seq.shift(); S.seq.pop(); }
    else { S.seq = []; }
} else {
    S.pnl -= ba; S.seq.push(b);
}
if (S.seq.length === 0) { S.phase = 'clear'; }

数式・計算仕様

初期数列[1, 1]（初回ベット: 1+1 = 2単位）
2in1の由来数列1サイクル = 2回分の損失を1セットで回収
ベット計算seq[0] + seq[末尾]（モンテカルロと同一）
稼働開始条件consecutive >= 2（待機中の連続負け）
数列クリア時phase = 'clear'（2連敗分の損失を回収）

メリット・デメリット

✓ メリット

  • 2連敗が稼働条件なので無駄な稼働を回避
  • 初期数列[1,1]で始まりベット急増を抑制
  • 数列クリアで2回分の損失を一括回収する設計

✗ デメリット・注意点

  • 待機フェーズでも損失が発生する（1単位×2）
  • 稼働後の連敗で数列が長くなる
