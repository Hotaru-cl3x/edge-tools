Barnett Edge

バーネット法をリアルタイム自動計算するツール（barnett-tool.js）。1→3→2→6単位という意図的なステップダウンを含む固定ラダーをSTEPS=[1,3,2,6]配列で実装。2連勝時に3→2へダウンして部分利益を確保してから最大ベット（6単位）に移行する独自設計。

ツールを無料で使う →

基本仕様

システム種別Positive Progression / Fixed Step with Step-down
対象ゲームバカラ・ルーレット（2倍配当）
ステップ配列STEPS = [1, 3, 2, 6]（step2でダウン）
4連勝以降6単位キープ
リスクMedium

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 意図的ステップダウン設計（barnett-tool.js）
var STEPS = [1, 3, 2, 6];
//           0  1  2  3  ← インデックス
// step1: 3単位 → step2: 2単位（意図的にダウン）

S = { si:0, streak:0, ... }

if (r === 'win') {
    S.pnl += ba;
    S.streak++;
    if (S.si < STEPS.length - 1) S.si++;
} else {
    S.pnl -= ba;
    S.si = 0; S.streak = 0;  // リセット
}

// ベット計算（4連勝以降は6単位キープ）
var si = Math.min(S.si, STEPS.length - 1);
var ba = STEPS[si] * S.u;
// si=0:1u, si=1:3u, si=2:2u(ダウン), si=3:6u

数式・計算仕様

ステップ配列STEPS = [1, 3, 2, 6]
step2のダウン3→2（2連勝時の意図的な賭け金減少）
4連勝以降6単位キープ
グッドマン（5u）との比較バーネットは最終ベット6単位（+1単位高い）
ステップダウンの意図中間利益を確保してからリスクを上げる設計

メリット・デメリット

✓ メリット

  • ステップダウンで中間利益を部分確保する独自設計
  • グッドマンより最終ベットが大きく連勝時の利益が高い
  • STEPS配列の変更だけで別バリアントに応用可能

✗ デメリット・注意点

  • step2のダウン（3→2）が直感に反しわかりにくい
  • 3連勝前の損失は全投資分（1+3+2=6単位）
