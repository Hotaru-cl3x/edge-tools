One Half Up Edge

ワンハーフアップ法をリアルタイム自動計算するツール（onehalfup-tool.js）。2連勝でシステム稼働し、以降の勝ちごとに0.5単位（half = u/2）を上乗せする最も緩やかな正順張りシステム。単位額には偶数を推奨（halfの整数化のため）。

基本仕様

システム種別Positive Progression / Incremental（+0.5単位/勝ち）
対象ゲームバカラ・ルーレット（2倍配当）
稼働条件2連勝（streak >= 2）
上乗せ額half = u / 2（1単位の半分）
リスクLow

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 段階的上乗せ（onehalfup-tool.js）
S = { half: u/2, betAmt: u, streak:0, ... }

if (r === 'win') {
    var ba = Math.floor(S.betAmt);
    S.pnl += ba; S.streak++;
    if (S.streak >= 2) {
        // 2連勝以降：前回ベット + 0.5単位
        S.betAmt = S.betAmt + S.half;
    }
    // streak=1: betAmtを変更しない（据え置き）
} else {
    S.pnl -= Math.floor(S.betAmt);
    S.streak = 0; S.betAmt = S.u;  // 1単位にリセット
}

// profit-lock表示（streak >= 2 の場合）
if (S.streak >= 2) {
    var safeAmt = S.pnl - Math.floor(S.betAmt);
    if (safeAmt > 0) { /* "今負けても残る利益: +X" 表示 */ }
}

数式・計算仕様

稼働後N連勝目のベット1 + (N-1) × 0.5 単位（N≥2から）
halfの計算S.half = S.u / 2
推奨単位額偶数（halfが整数になるため）
profit-lock条件streak >= 2 かつ pnl - nextBet > 0
上乗せペース比較イーストコースト（×1.5）の約1/3の速度

メリット・デメリット

✓ メリット

  • 全システム中で最も緩やかな上乗せペース（+0.5単位）
  • betAmtを実数管理し切り捨て誤差を防止
  • 2連勝後からprofit-lockが始まる安全設計

✗ デメリット・注意点

  • 上乗せペースが遅く大きな利益には長い連勝が必要
  • 1連勝時は据え置きで最初の連勝が利益に直結しない
