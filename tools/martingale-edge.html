Martingale Edge

マーチンゲール法をリアルタイム自動計算するツール。負けるたびに賭け金を2倍に更新し、損切りラインに到達する1つ前のベットで自動停止する安全機構を搭載。

ツールを無料で使う →

基本仕様

システム種別Negative Progression（逆張り型）
対象ゲームバカラ・ルーレット（2倍配当）
初期ベット1単位（固定）
リスクHigh
複雑度Low

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 状態オブジェクト（martingale-tool.js）
S = { u, m, c, bet:1, rd:0, pnl:0, hist:[], done:false }

// 勝敗処理
if (r === 'win') {
    S.pnl += ba;
    S.bet = 1;          // 1単位にリセット
} else {
    S.pnl -= ba;
    S.bet = S.bet * 2;  // 負けで2倍
}

// 損切りチェック（次ベットが限界を超えたら停止）
if (realLoss + ba > S.m) { S.done = true; }

数式・計算仕様

N連敗時ベット1 × 2^(N-1) 単位
N連敗の累計損失2^N - 1 単位
1勝の回収利益+1単位（常に固定）
損切り前の連敗限界cumLoss + 2^n <= stopLoss を満たす最大のn

メリット・デメリット

✓ メリット

  • 実装がシンプル（状態変数bet1つ）
  • 1勝で全損失＋1単位を回収
  • 損切りライン自動停止で暴走防止

✗ デメリット・注意点

  • 連敗でbetが指数的に増加（2^n）
  • テーブルリミットで回収不能になるリスク
  • 大きな元手が必要
