East Coast Progression Edge

イーストコーストプログレッション法をリアルタイム自動計算するツール（eastcoast-tool.js）。1連勝は据え置き、2連勝で3単位にジャンプ、3連勝以降は前回ベット×1.5（切り捨て）という動的乗数方式。betAmtを実数のまま内部管理して切り捨て誤差を最小化。

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 動的乗数方式（eastcoast-tool.js）
S = { betAmt:u, streak:0, prevSafe:0, safeCumul:0, ... }

if (r === 'win') {
    var ba = Math.floor(S.betAmt);  // 切り捨てで実際のベット額
    S.pnl += ba; S.streak++;

    if      (S.streak === 1) { /* 1連勝: 据え置き */ }
    else if (S.streak === 2) { S.betAmt = S.u * 3; }           // 3単位にジャンプ
    else                     { S.betAmt = S.betAmt * 1.5; }    // ×1.5（小数のまま保持）

    // safeCumul（確定利益累計）の更新（streak >= 3 から）
    if (S.streak >= 3) {
        var safeNow  = S.pnl - S.betAmt;
        var safeThis = Math.floor(safeNow - S.prevSafe);
        S.prevSafe   = safeNow;
        S.safeCumul += safeThis;
    }
} else {
    S.pnl -= Math.floor(S.betAmt);
    S.streak = 0; S.betAmt = S.u;
    S.prevSafe = 0; S.safeCumul = 0;
}

数式・計算仕様

streak=1のベット変化なし（1単位）
streak=2のベット3単位（固定ジャンプ）
streak=N（N≥3）のベットfloor(3 × 1.5^(N-2)) 単位
内部精度betAmtは実数のまま管理し表示・収支のみfloor()
profit-lock開始streak >= 3（safeCumulが正になった時点）

メリット・デメリット

✓ メリット

  • 動的乗数（×1.5）で固定ステップより急速に利益増大
  • betAmtを実数管理し切り捨て誤差を最小化
  • safeCumulで確定利益を精確に追跡

✗ デメリット・注意点

  • ×1.5の連鎖で3連勝以降のベット額が急増
  • streak=1は据え置きのため最初の2連勝は利益が小さい
