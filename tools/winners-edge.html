Winners Edge

ウィナーズ投資法をリアルタイム自動計算するツール（winners-tool.js）。2in1法と同じ2連敗トリガー式だが、ベット計算が「数列先頭の2倍」という独自方式。モンテカルロより攻撃的な回収速度を持つ。

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 2in1との決定的な違い（winners-tool.js）
// ベット計算：両端の和ではなく先頭×2
function bet() {
    if (S.phase === 'wait') return 1;
    var q = S.seq;
    if (q.length === 0) return 0;
    return q[0] * 2;  // ← 先頭の2倍（2in1: q[0]+q[末尾]）
}

// 勝敗処理（稼働フェーズ）
if (r === 'win') {
    S.pnl += ba;
    S.seq.shift();  // 勝ち→先頭を1つ削除（pop不使用）
} else {
    S.pnl -= ba;
    S.seq.push(b);  // 負け→ベット単位数を末尾に追加
}
if (S.seq.length === 0) { S.phase = 'clear'; }

// 待機フェーズは2in1と完全に同じ（2連敗でseq=[1,1]）

数式・計算仕様

ベット計算式seq[0] × 2（先頭値の2倍）
2in1との違い2in1: seq[0]+seq[末尾] / Winners: seq[0]×2
勝ち時の操作shift()（先頭のみ削除、末尾は操作しない）
初回ベット[1,1]の場合: 1×2 = 2単位
ベット増加ペース2in1より攻撃的（先頭が大きくなると倍増）

メリット・デメリット

✓ メリット

  • 先頭×2方式で2in1より攻撃的な回収速度
  • 同じ待機フェーズ設計で無駄な稼働を回避
  • 先頭のみ参照するシンプルな計算式

✗ デメリット・注意点

  • ベット額が2in1より大きくなりやすい
  • 深い連敗で数列が長くなりベット額も増大
