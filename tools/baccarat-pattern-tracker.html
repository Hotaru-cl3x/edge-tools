実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 罫線ストリーク抽出（baccarat-pattern-tracker.js）
function calcStreaks(arr) {
    var bp = arr.filter(r => r !== 'T');  // TIEを除外
    var streaks = [];
    for (var i = 0; i < bp.length; i++) {
        var r = bp[i];
        if (streaks.length === 0 ||
            streaks[streaks.length-1].type !== r) {
            streaks.push({ type: r, count: 1 });
        } else {
            streaks[streaks.length-1].count++;
        }
    }
    return streaks;  // 例: [{type:'B',count:3},{type:'P',count:2}]
}

// パターン検出（長さ2〜3の部分列で重複検索）
function detectPatterns(streaks) {
    var patternMap = {};
    for (var len = 2; len <= 3; len++) {
        for (var i = 0; i <= streaks.length - len; i++) {
            var slice = streaks.slice(i, i + len);
            var key = slice.map(s => s.type + s.count).join('-');
            patternMap[key] = (patternMap[key] || 0) + 1;
        }
    }
    // 2回以上出現したパターンのみ返す
    return Object.entries(patternMap)
        .filter(([k, v]) => v >= 2)
        .sort(([,a],[,b]) => b - a);
}

数式・計算仕様

パターンキー「出目タイプ+連数」の連結（例: B3-P2）
検索する部分列の長さ2〜3罫線単位
検出条件同一パターンが >= 2回出現
TIE出現率TIE数 / 全入力数 × 100%
グリッドサイズ6行 × N列（Nは動的に拡張）

メリット・デメリット

✓ メリット

  • 罫線単位の部分列マッチングによる高精度パターン検出
  • 大路グリッドをDOMで動的に描画
  • TIEを除外したcalcStreaks関数で純粋な勝敗ストリークを管理

✗ デメリット・注意点

  • パターン検出は統計的一致であり予測の保証ではない
  • シューが短い場合は比較可能なパターン数が少なくなる
