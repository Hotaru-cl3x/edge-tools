Fibonacci Edge

フィボナッチ数列ベースのベッティングシステム（内部実装名：cocomo-tool.js）。2連敗でシステム稼働し、以降フィボナッチ数列に従ってベットを増やす。3倍配当エリア（コラム・ダズン）専用設計。

基本仕様

システム種別Sequence-based / Negative Progression
対象ゲームルーレット 3倍配当エリア（コラム・ダズン）専用
フェーズ待機（1単位）→ 稼働（フィボナッチ数列）
稼働トリガー2連敗（waitLose >= 2）
リスクMedium

実装アルゴリズム

JavaScriptソースコードから抽出したコアロジックです。

// 2フェーズ制ロジック（cocomo-tool.js）
S = { phase:'wait', waitLose:0, seq:[], ... }

// 稼働時のベット計算（フィボナッチ: seq末尾2要素の和）
var bet = S.seq.length >= 2
    ? S.seq[S.seq.length-2] + S.seq[S.seq.length-1]
    : S.seq[0];

// 待機フェーズ
if (S.phase === 'wait') {
    if (r === 'win') {
        S.pnl += ba * 2;  // 3倍配当のため+2倍
        S.waitLose = 0; S.seq = [];
    } else {
        S.pnl -= ba; S.waitLose++;
        if (S.waitLose >= 2) {
            S.phase = 'active'; S.seq = [1, 1];  // 稼働開始
        }
    }
}
// 稼働フェーズ
if (r === 'win') {
    S.pnl += ba * 2;               // 3倍配当
    S.phase = 'wait'; S.seq = [];  // 勝ちで即リセット
} else {
    S.pnl -= ba; S.seq.push(bet); // 負けで次のFib値を追加
}

数式・計算仕様

フィボナッチ数列[1, 1, 2, 3, 5, 8, 13, 21, ...]
N番目のベット（稼働後）fib(N+1) 単位（seq末尾2要素の和）
3倍配当の収支勝ち: +ba×2 / 負け: -ba
稼働トリガーwaitLose >= 2
稼働終了トリガー稼働中に1勝（即時リセット）

メリット・デメリット

✓ メリット

  • 2連敗トリガーで不要な稼働を回避
  • 3倍配当に対応した収支計算（+ba×2）
  • 勝ち即リセットで稼働期間を最短化

✗ デメリット・注意点

  • 3倍配当専用のため汎用性が低い
  • 深い連敗でフィボナッチ数列が大きな値に到達
